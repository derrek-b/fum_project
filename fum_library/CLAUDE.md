# CLAUDE.md ‚Äî fum_library (Shared Utilities)

## What This Project Is

Shared library consumed by both `fum` (frontend) and `fum_automation` (automation service). Provides platform adapters, blockchain utilities, token/chain configs, contract ABIs, and helper functions. Installed via local tarball ‚Äî changes aren't visible to consumers until you run `npm run pack`.

## Commands

```bash
npm run build            # Build src/ ‚Üí dist/ (simple copy + docs generation)
npm run pack             # Build + pack tarball + install into fum & fum_automation
npm test                 # Vitest unit tests
npm run test:unit        # Unit tests only
npm run test:integration # Integration tests (requires Hardhat node)
npm run test:coverage    # Coverage report
```

**After any code change**: Always run `npm run pack` before testing in fum or fum_automation. They won't see your changes otherwise.

**Do NOT use npm link (sync/unsync)** to share fum_library with other projects. It causes issues with how the library is initialized across the ecosystem. Always use `npm run pack` for local development.

## Adapter Pattern

All DEX interactions go through adapters extending `PlatformAdapter` (base class in `src/adapters/PlatformAdapter.js`). The base class defines the interface with required methods that throw "must be implemented" and optional methods with safe no-op defaults.

**Required methods** (must override): `getPoolData`, `generateSwapData`, `generateCreatePositionData`, `generateRemoveLiquidityData`, `generateClaimFeesData`, `getAccruedFeesUSD`, `parseSwapReceipt`, `sortTokens`, `getOptimalTokenRatio`, etc.

**Optional methods** (override to enable): `getPoolIncentives`, `getIncentivePreCloseTransactions`, `getIncentivePostCreateTransactions`, `getIncentiveClaimTransactions`

**Current adapters**: UniswapV3Adapter, UniswapV4Adapter, TraderJoeV2_2Adapter

When adding a new platform: create a new adapter extending PlatformAdapter, implement all required methods, and register it in the adapter factory.

## Build System

The build is a simple file copy (`cp -r src/* dist/`), not a bundler. Source and dist are identical JS files. The `dist/` directory is what gets packed into the tarball and consumed by other projects.

## Test Infrastructure

The library exports test utilities for consumers:
- `fum_library/test/setup/hardhat-config` ‚Äî Shared Hardhat config and test accounts
- `fum_library/test/setup/test-contracts` ‚Äî Contract deployment helpers for test environments
- `fum_library/test/test-env` ‚Äî Test environment setup

These are used by fum_automation's test suite to spin up consistent test environments.

## Key Details

- **ethers.js v5** is a peer dependency ‚Äî all blockchain code uses v5 API
- Contract ABIs in `src/artifacts/` are auto-generated by `fum/scripts/extract-abis.js` ‚Äî do not edit manually
- `bytecode/` contains compiled contract bytecodes synced from fum_testing ‚Äî do not edit manually
- Token lists and chain configs in `src/configs/` are the source of truth for supported tokens and chains
